# ⚡ 《Grid Master: Cyber Network》v1.2 需求規格（第二輪遊玩回饋）

> 來源：2026-07-12 v1.1 上線後的遊玩回饋。四項皆為前端呈現層改動，零後端變更。

## R1 地圖線段與數字放大

線段寬度再加倍（網格 5→10px、地理 4→8px），過路費數字放大（15→20／14→18px）。
連動調整：網格佈局平行車道間距 0.08→0.12 格（≈21px，線寬 10px 下仍有明顯空隙）；
繞節點偏移隨之變為 ±0.48 格 ≈ 85px。真實地圖重新驗證可見共線重疊 = 0。

## R2 玩家數值卡整體放大一倍

`.player-chip`：min-width 170→300px、字級 0.85→1.15rem、內距與電廠標籤同步放大。

## R3 儲存資源凹槽顯示

玩家卡上以**凹槽**呈現儲存容量、**符號**呈現持有資源：

- 每種專屬容量（設施燃料需求 ×2）一列：`💧 [💧][💧][ ][ ]`。
- 混合容量另列（標籤 💧🔥），顯示水力／火力超出專屬容量的溢出——與引擎的
  總量驗證等價（engine-design §6.3），前端用 `capacity.js` 的 `capsOf` 直接算。
- 空凹槽用內陰影＋虛線框表達「可裝而未裝」。

## R4 採購階段的資源市場整合

- 採購資源階段，資源市場面板**移進操作面板的採購區塊**（所有人可見，含等待中
  玩家與旁觀者）；其他階段維持在下方面板列。
- **動態採購預覽**：調整數量 stepper 時，市場價格梯上「即將被買走的格位」
  （存量區最便宜的 n 格）即時變為琥珀色，右側顯示 `−n → $總價`——
  玩家直觀看到錢花在哪個資源的哪些價位上。
